## Что НЕ входит (сознательно отложено, не баги)

- **SIP-телефония (Mute / Hold / Transfer)** — заглушки с дисклеймером, нужен
  провайдер. В этой волне только «припарковали» (см. S1), сам SIP — отдельный
  проект.
- **AI-поверхности** (подсказки в звонках/чате, AI-rec в action-center) —
  выключены намеренно через `AI_ENABLED=false` + `InDevelopment` / `return null`.
  Это locked-решение, не баг (см. Приложение А).
- **Kiosk price `*100`** — рабочий воркэраунд под legacy-shape `/api/kiosk/doctors`.
  Чистка контракта API — отдельная backend-задача (см. Backlog B1).
- **`/api/crm/calls/active`** — отсутствующий live-эндпоинт колл-центра, новая
  backend-работа, вне freeze (Backlog B2).
- **Дефолт `enabled=false` в расписании отчётов** — продуктовый выбор, не баг
  (Backlog B3).

## Сделано в этой волне (реальные фиксы)

### S1 · SIP-заглушки больше не «тыкаются в тупик»

**Проблема.** `call-center/_components/call-actions-rail.tsx` — тайлы
Transfer / Mute / Hold были `disabled={!call}` + `onClick={onStub}`. Во время
активного звонка они становились кликабельными и по клику показывали тост
«Подключите SIP-провайдера…». То есть единственный реально видимый тупик: оператор
жмёт живую с виду кнопку и получает «недоступно».

**Решение.** Сделать SIP-тайлы **всегда `disabled`** — они видимы (оператор знает,
что появится после подключения SIP), но не кликабельны. Дисклеймер под блоком уже
объясняет почему; `title` оставлен как hover-подсказка. Удалён неиспользуемый
`onStub`. Реальные тайлы (Hangup / Mark missed) не тронуты — включаются по ходу
звонка как раньше.

**Acceptance.** Mid-call клик по Transfer/Mute/Hold ничего не делает (нет
тоста-тупика); `grep onStub` → 0; `tsc` + `eslint` по файлу зелёные. ✅

### S2 · Убраны мёртвые поля `telegram` / `callbacks` в action-center

**Проблема.** `action-center/_components/action-center-client.tsx` — тип `Buckets`
объявлял `telegram: ActionRow[]` и `callbacks: ActionRow[]`, а `bucketActions`
возвращал их всегда пустыми (`telegram: [] // wired later`). Верификация: эти поля
**нигде не читаются** (совпадения по «telegram» в файле — это `td("telegram")`,
`sla.data?.telegram` и href-строки, другой объект). Чистый мёртвый код.

**Решение.** Удалить оба поля из типа `Buckets` и из return `bucketActions`.

**Acceptance.** `tsc` зелёный (нет потребителей → ничего не ломается); поведение
страницы без изменений. ✅

### S3 · Демо-косметика прод-данных (вне кода, контекст)

Не код, а данные демо-инстанса neurofax: переименованы 4 мусорных доктора в живые
ФИО + назначены кабинеты, заполнена идентичность клиники (NeuroFax, телефон,
адрес). Сделано прод-операцией в прошлой сессии; здесь зафиксировано как контекст,
повторного деплоя не требует.

## Приложение А — опровергнутые находки аудита (НЕ баги)

Зафиксировано, чтобы не переоткрывать. Каждый пункт проверен вручную.

- **AI-подсказки в звонках** (`call-actions-rail.tsx`) — обёрнуты
  `<InDevelopment active={!AI_ENABLED}>`, корректно затемнены с бейджем «в
  разработке». By-design.
- **AI-rec / прочие AI-панели** — либо `if (!AI_ENABLED) return null` (скрыты),
  либо `InDevelopment`. Пустых «висящих» AI-блоков нет.
- **action-center `telegram`/`callbacks`** — аудит назвал «отображаются пустые
  массивы». На деле они **никогда не рендерились** — невидимый мёртвый код
  (удалён в S2), а не видимая пустота на экране.
- **Kiosk `price*100`** — не «сломанная цена», а рабочий воркэраунд (B1). Отдаёт
  корректные сумы.
- **Фабрикация пути** — аудит ссылался на `src/app/c/[slug]/kiosk/page.tsx`,
  которого не существует (есть только `src/app/kiosk/page.tsx`). Находка
  несостоятельна.

> Вывод: из «большого» списка аудита реальными оказались два пункта (S1, S2),
> оба закрыты. Раздувать скоуп ради видимости работы — против polish-only этики
> проекта.
